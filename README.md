# Veridis Rewrite

Veridis Rewrite 是一个面向 SillyTavern 的文本净化与局部改写扩展。它在传统屏蔽词替换的基础上，加入了规则预设、范围标签保护、净化前文透视，以及可选的 OpenAI 兼容 AI 局部改写能力，用来在聊天生成、历史记录和相关数据中稳定处理不想保留的词句、句式或表达。

当前版本：`7.0.0-alpha`

---

## 核心功能

### 规则净化

- 支持普通文本、正则表达式、简易通配三种规则模式。
- 支持一条规则配置多个目标词和多个替换候选；没有替换候选时会直接删除命中内容。
- 支持规则合集分组、单条规则启停、合集启停、排序、搜索和批量管理。
- 支持正则捕获组替换，例如 `$1`、`$2`，适合处理结构化文本。
- 支持简繁兼容词典，可按台湾、香港异体进行扩展匹配。

### AI 局部改写

- 支持 OpenAI 兼容接口，配置 Base URL、API Key、模型、温度、超时与重试次数后即可使用。
- AI 改写只处理命中 AI 改写规则的局部片段，不要求整段消息重写。
- 默认只扫描完整的 `<content>...</content>` 范围，避免把非正文区域误送给 AI。
- 支持按规则配置独立的 AI 提示词，让不同命中片段使用不同改写指令。
- 流式生成时可在 XML 闭合后提前请求 AI，并在最终净化阶段写回结果。
- 提供 AI 请求状态、终止按钮和诊断日志，方便排查未发送、超时、返回格式错误等问题。

### 实时净化

- 在消息生成、消息渲染、非流式生成结束、聊天切换等阶段自动执行净化。
- 支持视觉层实时遮罩与最终数据写回，减少流式输出时的漏屏、闪现和反复替换。
- 对长聊天采用分片后台处理，降低打开超长对话时的卡顿风险。
- 支持跳过用户消息，避免用户输入、用户消息编辑区和历史用户消息被规则误伤。

### 深度清理

- 可对聊天记录、聊天元数据、插件设置、角色卡、世界书和人设执行全域深度清理。
- 深度清理按阶段分片运行，并在固定时间窗口后询问是否继续，避免长时间卡死。
- 会跳过插件自身配置、内部净化元数据和已撤回净化的消息对象。
- 对数据库扩展字段做了保护，避免误清理 Prompt、Settings、Template 等配置内容。

### 净化前文透视

- 对最新若干条助手消息保留净化前后差异，可通过消息旁按钮查看。
- 支持片段模式和全文模式，片段模式聚焦变化位置，全文模式展示完整上下文。
- Diff 会区分删除、插入和 AI 改写来源，方便确认到底是哪条规则产生影响。
- 支持撤回净化，将当前消息恢复为原始文本，并进入免净化状态。
- 支持相关规则追踪，可从差异内容回查可能命中的规则并快速编辑。

### 范围标签保护

- 支持自定义范围标签，例如为指定标签包裹的内容跳过净化。
- 支持两种模式：保护标签内内容，或仅净化标签内内容。
- 内置标签可分组管理、折叠展示、启停和备注，便于处理思维链、状态栏、变量块等特殊文本。
- 支持保护用户人设描述，避免深度清理误改长期设定。

### 预设与绑定

- 支持多个净化预设，每个预设可以拥有独立规则和 AI 改写配置。
- 支持设置全局默认预设。
- 支持将当前净化预设绑定到角色，或绑定到 SillyTavern 当前对话补全预设。
- 支持规则合集在不同预设之间复制或移动。
- 支持预设 JSON 导入，并提供导入为新预设、导入并切换、临时预览等选择。

### 宿主与扩展兼容

- 兼容 SillyTavern 常规聊天、分支 Swipe、消息编辑、流式与非流式生成流程。
- 兼容 TauriTavern 的窗口化保存流程。
- 兼容柏宝箱后台生成保存窗口。
- 兼容 LoreFrame 拟界文库浮窗和楼层刷新。
- 保留 MVU 状态占位符，避免净化流程破坏状态栏更新链路。

## 使用入口

安装扩展后，SillyTavern 页面会出现两个入口：

- 顶部或工具区域的“词汇映射”按钮，用于打开 Veridis Rewrite 工作台。
- 扩展设置面板中的“屏蔽词净化助手 AI版”抽屉入口。

工作台内主要页面包括：

- “首页”：选择预设、查看规则数量和 API 检测状态。
- “AI”：配置 OpenAI 兼容接口、模型参数、请求限制和提示词模板。
- “净化”：管理范围标签、用户消息保护、透视按钮等净化行为。
- “工具”：执行深度清理、导入导出、诊断日志等维护操作。

---

## 基本流程

1. 打开“词汇映射”工作台。
2. 新建或选择一个净化预设。
3. 在规则合集里添加目标词、替换词和匹配模式。
4. 需要 AI 局部改写时，将对应子规则切换为“AI 改写”，并完成 AI 页面里的接口配置。
5. 保存预设。
6. 继续聊天，扩展会在生成和渲染过程中自动净化命中内容。
7. 如需检查处理结果，点击消息旁的净化前文透视按钮查看差异。

## 规则模式

### 普通文本

普通文本模式按字面量匹配目标词，适合稳定替换固定词汇。

### 简易通配

简易模式支持：

- `*`：匹配短范围内容，不跨越常见停顿符。
- `{A,B,C}`：匹配多个备选片段。

它适合处理句式模板，但比正则更容易维护。

### 正则表达式

正则模式支持裸 pattern，也支持 `/pattern/flags` 写法。系统会校验非法 flags、重复 flags、语法错误和空字符串匹配风险。

## AI 改写说明

AI 改写不是整篇润色，而是“命中片段的局部替换”。它会把完整消息作为文风参考，同时只要求模型返回命中片段对应的 JSON 改写结果。

使用 AI 改写前需要满足：

- 已启用 AI 改写。
- 已配置 Base URL、API Key 和模型。
- 助手消息中存在完整的 `<content>...</content>` 范围，或你已在设置里改为其他 XML 范围标签。
- 当前启用规则中存在“AI 改写”子规则，并且该规则在 XML 正文范围内命中。

如果没有发送 AI 请求，通常是配置不完整、目标消息不是助手消息、XML 标签未闭合、规则未命中或当前消息已撤回净化。

## 数据安全提示

- 深度清理会修改聊天记录、角色卡、世界书、人设等已有数据，请先确认当前预设和规则范围。
- 使用正则或简易通配时，避免写出可以匹配空字符串的规则。
- AI 改写会把命中范围内的上下文发送到你配置的接口服务，请确认接口来源和隐私边界。
- 如果需要保护特定内容，优先使用范围标签或“跳过用户消息”。
